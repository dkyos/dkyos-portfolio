# DKyos 포트폴리오 - PRD (Product Requirements Document)

## 1. 프로젝트 개요

개발자 기술 블로그 및 포트폴리오 사이트. 소프트웨어 개발, 웹 기술, AI 등 다양한 기술 주제의 글을 작성·발행하고, 이력 및 프로필을 소개한다.

### 핵심 가치

- **콘텐츠 중심**: 기술 글의 가독성과 시각 자료(Mermaid, Chart.js) 활용
- **관리 편의성**: 관리자 대시보드 + CLI 스크립트로 글 발행
- **SEO 최적화**: 구조화된 메타데이터, OG 이미지 동적 생성, RSS 피드

---

## 2. 현재 기능 명세

### 2.1 블로그

| 기능 | 설명 |
|------|------|
| 글 목록 | 공개된 글을 최신순으로 표시, 태그/카테고리/읽기 시간 포함 |
| 글 상세 | 마크다운 렌더링, Mermaid 다이어그램, Chart.js 차트, 코드 하이라이팅 |
| 조회수 | 글 조회 시 원자적 카운트 증가 (Supabase RPC) |
| 공유 | 클립보드 복사 (Web Share API) |
| RSS 피드 | `/feed.xml` 자동 생성 |

### 2.2 관리자 대시보드

| 기능 | 설명 |
|------|------|
| 인증 | Supabase Auth (이메일/비밀번호) |
| 글 작성/수정 | 마크다운 에디터 + 실시간 미리보기 |
| 글 삭제 | 확인 모달 후 삭제 |
| 공개/비공개 토글 | 즉시 상태 변경 |
| CLI 발행 | `scripts/publish-post.ts`로 마크다운 파일 일괄 발행 |

### 2.3 소개 페이지

| 기능 | 설명 |
|------|------|
| 프로필 | 이름, 역할, 연락처 |
| 타임라인 | 이력 및 교육 이력 (시각적 타임라인 UI) |

### 2.4 SEO

| 기능 | 설명 |
|------|------|
| 메타데이터 | 페이지별 title, description, OG 태그 |
| OG 이미지 | `/api/og`로 동적 생성 |
| JSON-LD | BlogPosting, Person 스키마 |
| Sitemap | 자동 생성 |
| robots.txt | 자동 생성 |

## 3. 향후 기능 로드맵

### 3.1 좋아요 (Phase 1)

- 익명 좋아요 (fingerprint 기반 중복 방지)
- 글 상세 페이지에 하트 버튼 + 카운트 표시
- 낙관적 업데이트 (즉시 UI 반영)

### 3.2 댓글 (Phase 2)

- 닉네임 + 내용 입력 (로그인 불필요)
- 관리자 승인제 (스팸 방지)
- 대댓글 지원 (1단계)
- 관리자 대시보드에서 댓글 관리 (승인/삭제)

### 3.3 검색 (Phase 3)

- 제목, 설명, 태그 기반 검색
- Supabase Full-Text Search 활용

---

## 4. 비기능 요구사항

### 성능

- ISR (Incremental Static Regeneration) 활용, 60초 revalidate
- 목록 조회 시 content 컬럼 제외 (네트워크 최적화)
- React cache()로 동일 요청 내 중복 호출 방지

### SEO

- Core Web Vitals 최적화
- 구조화된 데이터 (JSON-LD)
- 동적 메타데이터 + OG 이미지

### 접근성

- 시맨틱 HTML
- 키보드 네비게이션 (모달, 토글)
- 다크모드 지원

### 보안

- Supabase RLS (Row Level Security)
- 서비스 롤 키는 서버 전용
- API 인증 (Bearer token)
- 관리자 인증 (Supabase Auth + 쿠키)
